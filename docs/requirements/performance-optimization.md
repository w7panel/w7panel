## 一、背景与目标

### 1.1 问题背景

近期生产环境中 **kubelet 10250 端口频繁出现 503 错误**，导致以下功能受影响：

- 文件管理页面打开缓慢或失败
- 压缩/解压操作失败
- 权限修改操作失败
- 终端连接不稳定

### 1.2 优化目标

| 目标 | 指标 |
|------|------|
| 消除 10250 端口瓶颈 | 调用减少 90%+ |
| 提升 API 响应速度 | 平均响应时间 < 500ms |
| 减少 K8s API 负载 | 降低 50%+ 不必要请求 |
| 优化前端体验 | 首屏加载 < 3s |

---

## 二、现状分析

### 2.1 问题链路

```
用户操作                    触发 Exec                    K8s API
────────────────────────────────────────────────────────────────
打开文件管理        ───►  GetAgentPodAndPid()      ───►  10250 端口
                        │                                (crictl inspect)
                        ▼
刷新文件列表        ───►  GetAgentPodAndPid()      ───►  10250 端口
压缩文件           ───►  GetAgentPodAndPid()      ───►  10250 端口
权限修改           ───►  GetAgentPodAndPid()      ───►  10250 端口
```

### 2.2 Exec 触发场景

| 场景 | 触发条件 | 是否调用 exec | 影响程度 |
|------|----------|--------------|----------|
| **文件管理** | 每次打开页面 | ✅ 1次/会话 | 🔴 高 |
| **压缩/解压** | 每次操作 | ✅ 1次/操作 | 🟠 中 |
| **权限修改** | 每次操作 | ✅ 1次/操作 | 🟠 中 |
| **终端连接** | 新建连接 | ❌ WebSocket | 🟢 低 |
| **日志查看** | 直连 K8s | ❌ 无 | 🟢 低 |

### 2.3 根因分析

| 根因 | 影响 |
|------|------|
| **每次操作都获取 PID** | 大量重复 exec 调用 |
| **无本地内存缓存** | 无法跨请求复用 PID |
| **仅依赖 Pod Annotation** | 缓存失效后必须重新获取 |
| **Agent Pod 单点** | 所有请求汇聚，单点瓶颈 |

## 五、解决方案

### 5.1 核心方案：本地 PID 缓存

#### 5.1.1 架构设计

```
┌─────────────────────────────────────────────────────────────────┐
│                    优化后架构                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │              PID Cache Layer (本地内存)                 │  │
│  │  ┌────────────────────────────────────────────────────┐  │  │
│  │  │  Map<HostIP_ContainerID, PIDCacheEntry>          │  │  │
│  │  │  {                                               │  │  │
│  │  │    hostIP: "10.0.0.1",                           │  │  │
│  │  │    containerId: "containerd://xxx",             │  │  │
│  │  │    pid: 12345,                                   │  │  │
│  │  │    expiresAt: 1700003600                         │  │  │
│  │  │  }                                               │  │  │
│  │  └────────────────────────────────────────────────────┘  │  │
│  └──────────────────────────────────────────────────────────┘  │
│                              │                                  │
│                              ▼                                  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │              GetAgentPodAndPid()                         │  │
│  │                                                          │  │
│  │   1. Check Local Memory Cache  ──► 命中 → 直接返回    │  │
│  │      │                                                  │  │
│  │      ▼                                                  │  │
│  │   2. Check Pod Annotation    ──► 有效 → 写入缓存      │  │
│  │      │                                                  │  │
│  │      ▼                                                  │  │
│  │   3. Execute crictl (only if cache miss)               │  │
│  │                                                          │  │
│  └──────────────────────────────────────────────────────────┘  │
│                              │                                  │
│                              ▼                                  │
│                    ┌─────────────────────┐                     │
│                    │  kubelet:10250     │                     │
│                    │  (减少 90%+ 调用)  │                     │
│                    └─────────────────────┘                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 5.1.2 数据结构

```go
type PIDCache struct {
    mu         sync.RWMutex
    entries    map[string]*PIDCacheEntry
    defaultTTL time.Duration
}

type PIDCacheEntry struct {
    HostIP      string    // 节点 IP
    ContainerID string    // 容器 ID
    PID         int       // 进程 ID
    CreatedAt   time.Time // 创建时间
    ExpiresAt   time.Time // 过期时间
}
```

#### 5.1.3 缓存策略

| 参数 | 值 | 说明 |
|------|-----|------|
| TTL | 5 分钟 | 平衡新鲜度与性能 |
| 缓存键 | HostIP + ContainerID | 唯一标识 |
| 最大条目 | 10000 | 内存保护 |
| 淘汰策略 | LRU + TTL | 双重保护 |

### 5.2 备选方案：直接读文件

**说明**：不依赖 crictl 命令，直接读取容器运行时文件获取 PID。

| 运行时 | 文件位置 | 字段 |
|--------|----------|------|
| containerd | `/var/lib/containerd/.../config.json` | `info.pid` |
| docker | `/var/run/docker/containers/{id}/config.v2.json` | `State.Pid` |

**评估**：实现复杂度中等，但需要额外挂载 /var 目录，暂不推荐优先实施。

## 九、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 缓存数据过期 | PID 不准确 | TTL + 动态刷新 |

| 内存压力 | OOM | 最大条目限制 + LRU |
